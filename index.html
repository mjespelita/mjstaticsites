Why choose Static

Secure

The most prioritize aspect in web development especially after the site's deployment is the security. We all know that a site that is secure is the site that are more trusted. Most website on the internet are inevetably invade by an unknown attacker no matter how much security the site management put in. Because of my eagerness to protect user's data from this attackers, I decided to make all my projects static. Due to the absence of database which are phrone to attackers, your website will run smoothly without worrying on the bad consequences. You'll have peace mind.

Fast

Time is the most valuable thing on earth, ones it's gone, it's gone forever, so don't waste any of it waiting on slow sites. Static websites are the solution for a slow and annoying websites. Pages are already built with an static content written in a simple flat HTML file. Static websites are the FASTEST among database driven websites, interesting right?

Scalable

Static websites are easily to manage and update for it is only written in a plain text files. You can use any operating systems, devices you want anytime anywhere. Host on any server you want with cheaper cost. Some companies offer FREE hosting for static websites. Just run one command and your website is updated with new content and saves you a lot of time.
